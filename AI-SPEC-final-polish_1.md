# PART 1: หด Map (ตัดที่ว่าง ไม่ตัดห้อง) ✂️

## 1.1 ปัญหา
พื้นที่โล่งระหว่างห้อง/รอบนอกกว้างเกินไปหลายสิบ tile → เดินนาน, เกมยืด, Seeker กวาดตาทีเดียวเห็นหมด

## 1.2 กฎการหด (ห้ามทำผิดวิธี)

- ❌ **ห้าม scale ทั้ง map ลงแบบเหมารวม** — ห้องจะแคบ เฟอร์นิเจอร์ชนกัน
- ✅ **ขนาดภายในทุกห้องคงเดิม 100%** — ตัดเฉพาะ:
  1. ระยะห่างระหว่างห้อง → เหลือทางเดินกว้าง **3–4 หน่วย**
  2. ลานโล่งรอบนอกอาคาร → เหลือขอบไม่เกิน **2 หน่วย** จากผนังนอกสุด
- เป้ารวม: พื้นที่ map ลดลง **~30–40%**

## 1.3 เกณฑ์ตัวเลขที่ต้องคงไว้ (สูตร balance สำหรับ 10 คน)

| เกณฑ์ | ค่า | เหตุผล |
|---|---|---|
| Cover points | **คงเดิม 15 จุด** (ห้ามลด) | อัตราส่วน 1.5–2× ของ hider 8 คน |
| ทางเดินหลัก | ≥ 3 หน่วย | เดินสวน 2 คน + หลบ 1 |
| ประตูห้อง | 2 หน่วย | คงเดิม |
| ช่องระหว่างเฟอร์นิเจอร์ | ≥ 1.5 หน่วย | ไม่ติดขัด |
| ทางออกต่อโซน | **≥ 2 ทาง ทุกโซน** | ห้ามมีทางตันยาวทางเข้าเดียว (hider โดนต้อนจนมุม) |
| เวลา Seeker เดินตรวจครบ 15 จุด | **140–160 วิ** จาก 180 วิ | น้อยกว่านี้ = แคบไป, มากกว่า = ใหญ่ไป |

## 1.4 พื้นที่ว่างที่เหลือหลังหด
ถ้ายังมีลานโล่ง > 6×6 หน่วยตรงไหน ให้ถมด้วย props บังสายตา: กองกล่องเอกสาร, เครื่องถ่ายเอกสาร, ตู้ล็อกเกอร์แถว, ตู้น้ำดื่ม, ป้ายโปสเตอร์ตั้งพื้น — ให้ทุกเส้นทางมี "จังหวะบังตา" ไม่ใช่เห็นสุดลูกตา

# PART 2: ป้ายชื่อห้อง + Mission ระบุห้อง 🪧

## 2.1 ป้ายชื่อห้อง (ทุกห้อง 2 จุด)
1. **ป้ายเหนือประตู** ฝั่งทางเดิน: plane + `CanvasTexture` พื้นเข้ม ตัวหนังสือขาว สองภาษา เช่น "ห้องประชุม MEETING", "SERVER ROOM", "ห้องน้ำ TOILET", "PANTRY", "LOUNGE", "WORK ZONE A/B", "RECEPTION"
2. **ตัวอักษรจางบนพื้น** กลางห้อง: plane วางราบ opacity 0.12 ตัวใหญ่ (มุม isometric เห็นชัด)

## 2.2 Mission ต้องระบุห้อง
- เปลี่ยนข้อความ mission ทั้งหมดให้มีชื่อห้องต่อท้าย:
  - `SUBMIT THE REPORT` → `SUBMIT THE REPORT @ WORK ZONE A`
  - `CLEAR THE WHITEBOARD` → `CLEAR THE WHITEBOARD @ MEETING`
- จุดทำ mission มี marker เรืองแสงเล็กๆ (วงแหวน pulse บนพื้น) เห็นเมื่ออยู่ห้องเดียวกัน
- Minimap: จุด mission กะพริบสีเหลือง + สีห้องบน minimap ตรงกับสีพื้นห้องจริง

# PART 3: แสง + เฟอร์นิเจอร์เป็นชุด 💡

## 3.1 กลับด้าน contrast แสง (ตอนนี้สลับกันอยู่)
- **ธีม: office หลังเลิกงาน** — ทางเดิน = มืด (ไฟปิดแล้ว), ห้องที่ไฟเปิด = สว่างอุ่น
- ทางเดิน: โทนมืดน้ำเงินเข้ม + ไฟ emergency จุดเล็กๆ เป็นระยะ (แถบเรืองแสงเขียวจางเหนือประตู "EXIT" ก็ได้ — ฟีลดี)
- ห้องไฟเปิด: พื้น+ผนังสว่างโทนอุ่น `0xfff0d8`
- ห้องไฟปิด (จากสวิตช์): มืดเท่าทางเดิน → readable ทันทีว่าห้องไหนปิดไฟอยู่
- ผูกกับระบบสวิตช์ไฟเดิม — logic เดิมใช้ได้ เปลี่ยนแค่ visual

## 3.2 จัดเฟอร์นิเจอร์เป็น "ชุด" (แก้ของลอยเดี่ยว)
กฎ: **ห้ามมีเฟอร์นิเจอร์ชิ้นเดียวลอยกลางที่โล่ง** ทุกชิ้นต้องอยู่ในชุด:
- **ชุด cubicle** = โต๊ะ + จอมอนิเตอร์เรืองแสง (`emissive 0x2a4a6a`) + เก้าอี้ + partition ล้อม 2-3 ด้าน (ไม่ใช่ partition กากบาทเปล่าๆ)
- **ชุดโซฟา** = โซฟาหันเข้าโต๊ะกลาง + พรม + ทีวีติดผนังฝั่งที่โซฟาหัน
- ตู้/ชั้น → ชิดผนังเสมอ, ทีวี/whiteboard → ติดผนัง ไม่ลอยกลางห้อง
- Server room: rack เรียง 2 แถวเป็น aisle + ไฟ LED เขียว/ส้มกะพริบหน้าตู้ (สุ่ม toggle ทุก 0.5–1 วิ)

| Timer "176s" | แสดงเป็น **"2:56"** + ต่ำกว่า 30 วิ → สีแดง + pulse |

## 4.2 แสดง Ping ในเกม (ทำจริง — สำคัญสำหรับวัน event)
- ตัวเลข ping มุมล่างขวา อัปเดตทุก 2 วิ: วัด round-trip ผ่าน Colyseus (ส่ง `ping` message + timestamp, server echo กลับ)
- สี: เขียว < 80ms / เหลือง 80–150ms / แดง > 150ms
- ใช้แยกโรควัน event: "เกมกระตุก" → ดูตัวเลขรู้ทันทีว่าเน็ตคนนั้นหรือเกมเรา

## 4.3 ตรวจสถาปัตยกรรม Local Input (ต้องยืนยัน)
- **Local player ต้องขยับทันที 0ms ที่กดปุ่ม** ไม่รอ server ตอบ (server validate ทีหลัง)
- วิธีทดสอบ: DevTools → Network → Throttling "Slow 3G" → ตัวเอง**ยังขยับทันที** (คนอื่นกระตุกได้ ไม่เป็นไร) = ผ่าน
- ถ้าตัวเองหน่วงตาม = โค้ดรอ server ก่อนขยับ → **ต้องแก้** เป็น client-move-first

## 4.4 Performance budget
- FPS ≥ 55 บนโน้ตบุ๊กออฟฟิศ (ทดสอบเครื่องอ่อนสุดของทีม)
- ถ้าตก: ลด shadow.mapSize → 1024, จำกัด castShadow เฉพาะกำแพง+เฟอร์นิเจอร์ใหญ่, รวม static mesh (merge geometry) ลด draw call
